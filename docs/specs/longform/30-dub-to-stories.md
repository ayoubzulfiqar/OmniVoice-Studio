# Implementation Spec — TASK #30: Dub → Stories cast+lines export ("Create Story from Dub")

## TL;DR
Add a "Create Story from Dub" action to the Dub editor header that takes the current `dubSegments` + `speakerClones`, groups segments by `speaker_id` into a Stories **cast** (one `CastMember` per speaker, voice = that speaker's assigned `profile_id`), maps each segment `{start, end, text, speaker_id, profile_id}` → a Stories `StoryTrack` `{character, text, profileId}`, loads them into the Stories Editor, and navigates there (`setMode('stories')`). Pure transform lives in a new util `frontend/src/utils/dubToStory.js`; one new store action `loadStoryFromDub` in `frontend/src/store/storiesSlice.ts`; one button in the active-editor header of `frontend/src/pages/DubTab.jsx`. No backend changes.

> **No API / SSE / DB surface (read this before hunting for an endpoint).** This feature is **100% client-side**: there is **no new HTTP request/response, no SSE event type, no WebSocket frame, no backend route, no Python code, no DB table, no alembic migration, and no Zustand-persist `version` bump.** The only "data store" touched is the existing `localStorage` blob `omnivoice.app` (persist `version: 4`), and only through shapes it already persists (`cast`, `storyTracks`, `currentProjectId`). The complete data contract is pinned in **§"API / data shapes"** below — a developer can implement every function from the signatures there without reading any other file. The only network-adjacent symbols in the touched code (`API`, `/longform/render`, `/dub/audio/:jobId`) are **pre-existing and unchanged** — this transform never calls them; downstream Stories render uses them later, untouched by this PR.

## Problem
The Dub pipeline produces a fully attributed, multi-speaker transcript (`dubSegments`, each with `speaker_id`, `text`, `start`, `end`, and an assigned `profile_id`), but that work is locked inside the dubbing workflow. Users who want to re-voice the same conversation as a multi-track audiobook/story have to re-type every line and re-assign every voice in the Stories Editor by hand. The Stories Editor already models exactly this shape (cast = characters→voices, tracks = ordered lines — see `StoryTrack`/`CastMember` at `storiesSlice.ts:12-26`), and already renders through the shared `/longform/render` pipeline. The bridge between the two is missing.

## Goal / Non-goals
**Goals**
- One-click "Create Story from Dub" from the Dub editor when segments exist.
- Group `dubSegments` by `speaker_id` → distinct Stories cast members, deterministic order (first-appearance).
- Map each segment to a `StoryTrack` preserving order (by `start`), text, and resolved voice.
- Resolve each speaker's voice from the per-speaker `profile_id` assigned in the Dub Cast UI (`DubTab.jsx:831-867`); carry auto-clone selections (`auto:<speaker>`) into a real `profileId` when a saved clone profile exists, else fall back to cast default.
- Navigate to the Stories tab with the new cast + tracks loaded; do not silently clobber unsaved Stories work (confirm-or-merge — see Design).
- Pure, unit-tested transform; i18n for all user-facing strings.
- **Total robustness against malformed/partial segment data** — never throw, never produce a Story that crashes the StoriesEditor renderer (see Edge cases & failure paths). This robustness is **load-bearing, not defensive polish**: `DubSegment` types every field except `id`/`text` as `unknown` (`dubSlice.ts:46`), so the transform receives genuinely untyped data and the coercion rules in §"API / data shapes" are the contract.

**Non-goals**
- No translation choice surfaced here — we export the **target/edited** `text` (i.e. what the user sees in the table). (A future toggle for original vs translated is noted under Risk.)
- No timing/`start`/`end` carry-over into Stories (Stories has no timeline; `start` is used only for ordering). `pause_ms` synthesis from gaps is out of scope.
- No backend endpoint, no HTTP/SSE shape, no schema change, no persistence migration beyond what `storiesSlice` already does.
- No reverse direction (Story → Dub) — that is TASK #24/#31.

## Design

### Data flow
```
dubSegments (store) ──┐
profiles (prop) ──────┼──► dubToStory(segments, { profiles, unknownSpeakerLabel })
                      ┘            │

                                   ▼
                    storiesSlice.loadStoryFromDub(cast, tracks)
                                   ▼
                    setMode('stories')  ──► StoriesEditor renders
```
*(`speakerClones` is intentionally dropped from the v1 transform signature — see E24 / §"API / data shapes". The call site still has it available as a prop should a future version re-add it.)*

### 1. Pure transform — `frontend/src/utils/dubToStory.js` (NEW)

**Exact exported signature** (see §"API / data shapes" for the full JSDoc/TS typedef block to copy into the file):
```js
export function dubToStory(segments, { profiles = [], unknownSpeakerLabel = 'Speaker' } = {})
  // → { cast: CastMember[], tracks: StoryTrack[], stats: { lines: number, speakers: number, skippedEmpty: number } }
```
Algorithm:
1. **Input guard.** If `segments` is not an array (`Array.isArray(segments) === false` for null/undefined/object/string/number), treat it as `[]` and return the empty result (`{ cast: [], tracks: [], stats: { lines: 0, speakers: 0, skippedEmpty: 0 } }`). Likewise coerce `profiles` to `[]` if a caller passes a non-array (`Array.isArray(profiles) ? profiles : []`). The function must never throw on bad input.
2. **Filter to renderable segments.** For each segment compute `txt = (typeof seg?.text === 'string' || typeof seg?.text === 'number') ? String(seg.text).trim() : ''`. Keep a segment only when `txt !== ''`. Count every dropped segment (empty, whitespace-only, missing `text`, non-string/non-number `text`, or `seg` itself non-object) as `skippedEmpty`. (Note: numeric `text` like `0` survives because `String(0).trim() === "0"`; `null`/`undefined`/objects/arrays → `''` → dropped.)
3. **Sort defensively.** Sort a shallow copy by numeric `start` ascending, **stable**. Normalize via `const n = Number(seg?.start); const start = Number.isFinite(n) ? n : 0` so `undefined`/`null`/`NaN`/non-numeric-string `start` all collapse to `0` and keep their relative input order (stable sort preserves first-appearance as the tiebreak). The Dub table is already start-ordered, but SRT-imported or externally-merged segments may not be. Negative `start` is valid and sorts before `0` — acceptable, ordering is purely relative.
4. **Collect distinct speakers in first-appearance order.** Walk the sorted, filtered segments; derive a normalized speaker key per segment:
   - `rawSpk = seg?.speaker_id`; coerce to string: `spk = (rawSpk == null) ? '' : String(rawSpk)`.
   - If `spk.trim() === ''` (falsy, whitespace-only, or a non-string that stringifies empty) → map to the **single synthetic speaker** with key `' __unknown__'` (an internal sentinel containing a NUL byte that can never collide with a real `speaker_id`), display name `unknownSpeakerLabel`. All unattributed segments share this one synthetic speaker (do **not** create one synthetic speaker per unattributed segment).
   - Otherwise the speaker key is the trimmed `spk`.
   Collect keys in first-appearance order into an ordered list (`Array<string>`).
5. **Build cast.** For each distinct speaker key, in first-appearance order, create:
   ```js
   { id: castIdFor(displayName, assignedIds), name: displayName, color: nextCastColor(accum), profileId: resolveVoice(firstSegmentForSpeaker, displayName, profiles) }
   ```
   where `displayName` is the trimmed real `speaker_id`, or `unknownSpeakerLabel` for the synthetic speaker, `accum` is the cast array built so far (so `nextCastColor` sees prior picks), and `assignedIds` is a `Set<string>` of ids already minted in this cast pass.
   - `castIdFor(name, assignedIds)` — exact signature/return in §"API / data shapes". Slugify with the **same rule as `StoriesEditor.slug()` (`StoriesEditor.jsx:173`)** — `name.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '') || 'char'`. **This regex is intentionally identical to the live `slug()` and is linear-time** (`[^a-z0-9]+` is one bounded negated char class with no nested/overlapping quantifiers — no `(a+)+`, no `\s*…\s*` overlap), so it stays clear of CodeQL `js/polynomial-redos` even though it runs on user/model-derived `speaker_id` input (see Constraints). **Special case** (mirror `autoCast` at `StoriesEditor.jsx:184`): if `name.toLowerCase() === 'narrator'`, the base id is exactly `narrator` (so it aligns with the always-present default narrator and `deleteCharacter`'s `id === 'narrator'` guard at `StoriesEditor.jsx:166`). Then **guarantee uniqueness** within this cast: with `base` as the slug, if `assignedIds.has(base)` append `-2`, `-3`, … until free; add the chosen id to `assignedIds`; return it. Collisions are real and must be handled:
     - two different `speaker_id`s that slug identically (`"Speaker 1"` vs `"Speaker  1"`, `"a/b"` vs `"a-b"`);
     - a `speaker_id` that slugs to empty (e.g. all-punctuation `"???"`) → base `'char'`, then `'char-2'` for a second such speaker;
     - a real speaker literally named "Narrator" alongside the synthetic-unknown speaker whose `unknownSpeakerLabel` localizes to "Narrator" — the second one takes `narrator-2` (the `narrator` special-case only claims the id for the *first* narrator-slug speaker).
   - Maintain a `speakerKey → castId` map (`Object` or `Map<string, string>`) for the track pass.
   - `nextCastColor(accum)` from `frontend/src/utils/storyCast.js:15-18` — **exact semantics:** returns `CAST_COLORS.find(c => !usedSet.has(c))` (first palette color not already in `accum`), and **only when all 8 are used** falls back to `CAST_COLORS[(accum.length || 0) % 8]`. So for ≤8 speakers each gets a distinct color; the 9th+ deterministically reuse by index. (Not "wrap by count" for the first 8 — it's "first unused".) The palette is `['#fabd2f','#d3869b','#83a598','#b8bb26','#fe8019','#8ec07c','#fb4934','#458588']` (`storyCast.js:9-12`).
   - `resolveVoice(seg, displayName, profiles)` — see step 6 — is called on the **first segment** of each speaker (first by the sorted order). Within a clean dub job every segment of a speaker shares one `profile_id` (the Cast UI writes it to all of them, `DubTab.jsx:845`), but reading first-seen is defensive against SRT-imported segments where they could differ; the **first** segment wins and the divergence is silently ignored (documented under Edge cases).
6. **`resolveVoice(seg, displayName, profiles)`** — exact signature/return in §"API / data shapes". Resolve a segment's `profile_id` to a Stories `profileId | null`:
   - `pidRaw = seg?.profile_id`; coerce: `const pid = (pidRaw == null) ? '' : String(pidRaw)`.
   - `pid === ''` (empty / missing / falsy after coercion) → `null` (Stories "Default").
   - `pid.startsWith('auto:')` (via `startsWith`, **not** regex) → it's an ephemeral auto-clone selection. Attempt to bind it to a **saved** profile so it survives the job:
     - The match key is the speaker's `displayName` (the original `speaker_id`), **not** the `auto:` payload. (The payload is the underscored slug ``auto:${spk.toLowerCase().replace(/\s+/g,'_')}`` from `DubTab.jsx:836` — lossy, so don't reverse it.)
     - Find the first `profiles[i]` where `String(profiles[i]?.name ?? '').trim().toLowerCase() === displayName.trim().toLowerCase()`. If found → `profiles[i].id`. **Multiple profiles with the same name:** first match wins (deterministic by `profiles` array order). If none → `null` (degrade to cast Default — never dangle).
     - Edge: `unknownSpeakerLabel` speaker carrying an `auto:` value matches a profile literally named "Speaker" only if one exists; otherwise `null`.
   - `pid.startsWith('preset:')` (via `startsWith`) → `null` for v1. Design presets (`DubTab.jsx:856-858`) are not `profileId`s the Stories voice dropdown lists; mapping to `null` avoids a permanently-blank dropdown. Noted in Risk.
   - any other non-empty string → treat as a real saved `profileId`, **pass through verbatim** (`return pid`). (From the `dub.clone_profiles` optgroup, `value={p.id}`, `DubTab.jsx:851-855`.) Do **not** validate against `profiles` — passing an unknown id through is safe (the Stories voice `<select>` renders blank/Default for an unknown id; `effectiveProfile` returns it and `/longform/render` falls back to the default voice). Tolerates a profile deleted between dub-assignment and story-creation.
   - **Regex-safety note:** all four discriminations (`''`, `auto:`, `preset:`, pass-through) use equality + `startsWith`, **no regex on the user-controlled `profile_id`**. The only regex in the whole transform is the bounded linear `slug()` in step 5.
7. **Build tracks.** For each filtered+sorted segment, in order:
   ```js
   { id: ++localId, character: castIdMap[speakerKey], text: txt, profileId: null, emotion: null, speed: null }
   ```
   - `txt` is the same trimmed string computed in step 2 (`String(seg.text).trim()`), already known non-empty.
   - `localId` is a `1..N` sequence local to this call (`let localId = 0;` start, pre-increment → first id is `1`).
   - `profileId` is `null` (per-line override unset) — voice inherited from the cast member, matching `effectiveProfile(track, cast)` (`storyCast.js:21-25`: line override → cast voice → default). `emotion`/`speed` are `null`.
   - `character` is the cast id resolved from the segment's speaker key via `castIdMap` — guaranteed present because the cast pass walked the same filtered+sorted segments first, so every track's `character` references an existing cast member (no orphan tracks). An orphan `character` would make StoriesEditor render the line under a missing cast member and fall to default voice with no way to fix it.
8. Return `{ cast, tracks, stats: { lines: tracks.length, speakers: cast.length, skippedEmpty } }`.

Keep the util free of React/i18n: the caller passes a resolved `unknownSpeakerLabel` string. (This is also what keeps the transform out of the hardcoded-CJK gate — no `t()` call, no literal CJK; the localized label is injected by the caller — see Constraints.) The StoriesEditor mount effect re-seeds its module-level `_trackId` from `max(track.id)` on load (`StoriesEditor.jsx:136-140`), so lines added after load won't collide with the `1..N` ids minted here.

### 2. Store action — `frontend/src/store/storiesSlice.ts` (`loadStoryFromDub`)
Add to the `StoriesSlice` interface (`storiesSlice.ts:36-51`, alongside `loadProject`/`newProject`):
```ts
loadStoryFromDub: (cast: CastMember[], tracks: StoryTrack[]) => void;
```
Add the implementation in `createStoriesSlice` (`storiesSlice.ts:66-114`, next to `loadProject` at 101-105 and `newProject` at 106):
```ts
loadStoryFromDub: (cast, tracks) =>
  set({
    storyTracks: (tracks || []).map((t) => ({ ...t })),
    cast: (cast || []).map((c) => ({ ...c })),
    currentProjectId: null,
  }),
```
- Mirrors `newProject()` semantics (fresh, unsaved project) + load — `currentProjectId: null` so a subsequent `saveProject(name)` (`storiesSlice.ts:84-100`) creates a **new** named project rather than overwriting (it only updates in place when `currentProjectId` is set; `id = s.currentProjectId || genProjectId()` at line 86).
- Defensive `|| []` so the action never throws if called with `undefined`. Defensive shallow copy on both arrays (consistent with `loadProject` at line 104) so the caller's arrays can't mutate store state afterward. **Note the copy is one level deep** (`{ ...t }` / `{ ...c }`) — matching `loadProject`; track/cast member fields are all primitives or `null`, so a shallow copy fully isolates them.
- **Backward-compat (no migration).** The action sets exactly the persisted shapes `storyTracks` (`{id,character,text,profileId,emotion,speed}`, destructured identically in `partialize` at `store/index.ts:109-110`) and `cast` (`index.ts:111`). It introduces **no new persisted key and no shape change**, so the persist `version` stays `4` and the existing pass-through `migrate` (`index.ts:120-130`) is untouched. Old `omnivoice_data`/`omnivoice.app` localStorage rehydrates exactly as before.
- **Empty-cast edge:** if `cast` is `[]` (only if a caller ignores the empty-tracks guard), the store ends with `cast: []`, `storyTracks: []`. StoriesEditor's narrator-default (`DEFAULT_CAST`) is **not** re-injected by this action — `loadStoryFromDub([], [])` leaves an empty cast, unlike `newProject()` which seeds `DEFAULT_CAST`. The UI guard (step 3) prevents this in practice. (Decision: do **not** seed `DEFAULT_CAST` here — a dub-derived story always has ≥1 real cast member by the time it's called.)
- **Re-entrancy note:** the StoriesEditor `_trackId` reseed effect runs only on *mount* (empty dep array, line 139). If a Story is already open when the user runs Create-Story-from-Dub a second time, the effect does **not** re-run, so `_trackId` retains its prior (higher) value — new lines added after the second load still get ids above the loaded `1..N`, so no collision. The module counter being larger than `max(loaded id)` is harmless (ids only need uniqueness, not contiguity).

### 3. UI entry point — `frontend/src/pages/DubTab.jsx`
Add a "Create Story from Dub" button in the **active-editor** header actions — the `dub-head__actions` block at **`DubTab.jsx:715-750`** (inside the post-transcription `dub-col` rendered when `dubJobId && (dubStep === 'editing' || dubStep === 'generating' || dubStep === 'done')`, line 704). Do **not** use the idle skeleton header at line 357 (its buttons are `disabled`).

**Visibility & disabled states (enumerated):**
- **Hidden** entirely when `!dubSegments.length`. Because the whole `dub-col` only renders for `dubStep ∈ {editing,generating,done}`, the button can never appear before transcription.
- **Visible + enabled** when `dubSegments.length > 0` and `dubStep ∈ {editing, done}`.
- **Visible + disabled** when `dubStep === 'generating'` or `dubStep === 'stopping'` — generation is mutating segments/progress; block the hand-off until it settles. (Mirror the existing primary-row gating at `DubTab.jsx:724-728`.)
- The transform reads `dubSegments` as they currently are; if the user edited text in the table, the latest edits are exported (store is the single source of truth).

Handler (new `onCreateStory`, defined with `useCallback` like the existing `onGenerateClick` at `DubTab.jsx:197`):
```js
import { dubToStory } from '../utils/dubToStory';
// store reads (add near the existing reads at DubTab.jsx:101-110):
const setMode           = useAppStore(s => s.setMode);           // from uiSlice (uiSlice.ts:94)
const loadStoryFromDub  = useAppStore(s => s.loadStoryFromDub);  // from storiesSlice (new)
const storyTracks       = useAppStore(s => s.storyTracks);        // for the clobber guard

const onCreateStory = useCallback(() => {
  if (!dubSegments.length) return;                                   // guard A: nothing to export
  const { cast, tracks, stats } = dubToStory(dubSegments, {
    profiles,
    unknownSpeakerLabel: t('stories.fromDub.unknownSpeaker'),
  });
  if (!tracks.length) { toast.error(t('dub.create_story_empty')); return; }  // guard B: every line empty/whitespace
  // guard C — clobber: an existing unsaved/loaded story would be replaced.
  if (storyTracks.length && !window.confirm(t('dub.create_story_confirm'))) return;
  loadStoryFromDub(cast, tracks);
  setMode('stories');
  toast.success(t('dub.create_story_done', { lines: stats.lines, speakers: stats.speakers }));
}, [dubSegments, profiles, storyTracks, loadStoryFromDub, setMode, t]);
```
**Handler control-flow, fully spelled out:**
- **Guard A** — `dubSegments.length === 0`: return silently (button is hidden here, but the guard protects against a stale-closure click during a tab transition).
- **Guard B** — transform returned `tracks.length === 0` (all segments empty/whitespace/malformed): show `dub.create_story_empty` toast, do **not** navigate, do **not** call `loadStoryFromDub` (preserves an existing Story). Distinct from guard A: there were segments, but none produced a line.
- **Guard C** — clobber: only prompt when `storyTracks.length > 0`. **Cancel** → return without changing anything (no navigation, no store write, no toast). **Confirm** → proceed. When `storyTracks.length === 0` (fresh Stories, or `newProject` state), skip the confirm and proceed directly. A loaded **saved** project being edited still has `storyTracks.length > 0`, so the confirm fires — replacing it does **not** delete the saved project from `storyProjects` (it only swaps the live working tracks/cast and sets `currentProjectId: null`); the user can reopen the saved one. The confirm copy makes clear it replaces the *current working* project, not delete saved ones.
- **Success path**: `loadStoryFromDub` → `setMode('stories')` → success toast with `{lines, speakers}`. `skippedEmpty` is **not** surfaced in v1; it's in `stats` for a future "N empty lines skipped" note.
- **Double-click / rapid re-click:** idempotent per-click; a second click after navigation re-runs the same transform on the unchanged `dubSegments` and prompts the clobber confirm (since `storyTracks` is now populated). Acceptable; no debounce.

## Risk
- **Auto-clone voices don't survive the hand-off.** `auto:<speaker>` profiles are per-dub-job artifacts (`speakerClones`, `DubTab.jsx:837`), not saved voice profiles. A Story referencing them would dangle once the dub job is gone, so v1 degrades them to `null` (cast Default) unless a saved profile with the same name exists in `profiles`. *Mitigation:* note in a success toast or a follow-up "save clone as profile" (ties into TASK #25). Low risk; dangling refs are worse. (E18–E20.)
- **Original vs translated text.** v1 exports the current `text` (translated/edited). A user who translated the dub gets a target-language story; one who didn't gets source language. *Mitigation:* acceptable default ("what you see is what you get"); a future toggle could offer `text_original`. Documented as Non-goal.
- **`preset:` voices dropped.** Design-preset assignments (`DubTab.jsx:858`) map to `null` (E21). Low impact — presets aren't first-class Stories voices.
- **Diverging `profile_id` across one speaker's segments** (SRT-import path). First-seen wins; later assignments silently ignored (E16). Low risk — the in-app Cast UI writes uniformly.
- **Clobbering an in-progress Story.** Guarded by `window.confirm` when `storyTracks.length > 0` (E28–E30). The new project is unsaved (`currentProjectId: null`) so saved projects in `storyProjects` are never overwritten silently (E29). **Test coverage gap (accepted):** the confirm/navigation path is manual-smoke only, not unit-tested (the data effect it triggers — `loadStoryFromDub` + new-project-on-save — *is* slice-tested). If this path regresses, the smoke catches it, not CI.
- **Empty-cast defensive state.** `loadStoryFromDub([], [])` leaves an empty cast (no narrator re-seed). Not user-reachable (UI guard B blocks `tracks.length===0`), but unit-tested (`empty cast is not re-seeded with DEFAULT_CAST`) so a future refactor can't regress it into a silent empty-Stories crash.
- **Slug collisions across speakers** — handled by uniqueness suffixing in `castIdFor`; covered by tests E12–E15 (`suffixes colliding slugs…`, `empty-slug…`, `a speaker named Narrator…`, `unknownSpeakerLabel "Narrator"…`).
- **i18n missing-key regression (no automated catch).** There is no CI key-parity gate, so a forgotten key in one of the 20 non-English locales ships a raw `dub.create_story_done` to those users. *Mitigation:* the mirroring checklist + `scripts/translate_all.py`; a future tiny vitest parity test would close this (noted as optional hardening).
- **Constraint-regression risk (low, test-guarded).** A future edit to the slug regex (e.g. adding `\s*` overlap) could introduce ReDoS and trip CodeQL; the `adversarial-length speaker_id` timeout test + "copy slug verbatim" note guard against it. A future contributor adding a hardcoded confirm/toast string instead of a `t()` key would trip the CJK gate; the all-strings-via-`t()` rule and 21-file mirroring checklist guard against it.
